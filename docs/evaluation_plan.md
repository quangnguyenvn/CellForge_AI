# CellForge AI Local Evaluation Plan

This pack covers two local-first engineering tracks:

1. Evidence auditor eval set
2. Retrieval benchmark

Both are designed to run on a normal laptop without GPU. They are intentionally small and deterministic so you can improve them before spending Google Cloud credits.

## Why This Matters

CellForge AI should not be judged only by whether a generated hypothesis sounds plausible. It should be judged by whether the system can retrieve the right evidence, avoid misleading near-matches, cite supporting claims, surface contradictions, and lower hallucination risk.

## Retrieval Benchmark

Run:

```bash
python scripts/run_retrieval_benchmark.py
```

Metrics:

- `paper_recall_at_5`: expected papers found in top 5
- `claim_recall_at_10`: expected claims found after retrieving top papers
- `paper_mrr`: reciprocal rank of first expected paper
- `claim_mrr`: reciprocal rank of first expected claim
- `hard_negative_rate_at_5`: how often misleading papers appear in top 5

Good first targets:

```text
paper_recall_at_5 >= 0.70
claim_recall_at_10 >= 0.60
hard_negative_rate_at_5 <= 0.25
```

## Report Artifacts

Generate machine-readable and demo-friendly reports:

```bash
python scripts/generate_eval_report.py

python scripts/update_eval_baseline.py
```

Outputs:

```text
reports/eval_report.json
reports/eval_report.md
reports/eval_baseline.json
```

The Markdown report is meant for demos. The JSON report is meant for CI, Phoenix logging, or future comparison across retrieval/auditor implementations.

## Adapter Mode

The current runnable adapter is `local_mock`:

```bash
python -m services.evals.local_eval all --adapter local_mock
```

The reserved `pipeline` adapter is intentionally present as an integration point. Once the full CellForge API/pipeline code is restored, wire that adapter to the real retrieval and evidence-auditor services so the same 200-case harness can evaluate production behavior.

Each case includes:

- a goal
- a hypothesis
- supporting claim IDs
- contradicting claim IDs
- expected score thresholds
- expected recommendation

Audit dimensions:

- citation coverage
- claim grounding
- contradiction coverage
- hallucination risk
- recommendation

## How To Extend

The current scaffold starts with 200 retrieval cases and 200 evidence-auditor cases. The first 30 in each file are seed cases; the rest are deterministic expansion variants generated by `scripts/generate_eval_case_expansion.py`.

Regenerate the 200-case benchmark:

```bash
python scripts/generate_eval_case_expansion.py
```

Add more JSONL rows or generator templates when you want to push coverage toward a larger benchmark:

```text
Good hypotheses
Weak citation hypotheses
Overclaim / domain-transfer hypotheses
Missing contradiction hypotheses
Hard-negative retrieval queries
Cross-lever retrieval queries
```

For hackathon narrative, show this message:

> CellForge AI is not just generating hypotheses. It is building an evaluation loop around retrieval, evidence grounding, contradiction coverage, and hallucination risk.
